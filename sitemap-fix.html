GitHub Pages Sitemap Fix

Step-by-step guide to resolve sitemap issues

1

Verify your files exist

First, make sure your sitemap.xml and robots.txt files are in the root directory of your GitHub repository.

Visit these URLs to check if they are accessible:

  • https://srivenkateswaraborewells.github.io/WebSite/sitemap.xml
  • https://srivenkateswaraborewells.github.io/WebSite/robots.txt
2

Create proper robots.txt

Your robots.txt file should contain:

robots.txt
User-agent: * Allow: / Sitemap: https://srivenkateswaraborewells.github.io/WebSite/sitemap.xml
3

Create proper sitemap.xml

Your sitemap.xml file should follow the correct format:

sitemap.xml
<?xml version="1.0" encoding="UTF-8"?> <urlset xmlns="http://www.sitemaps.org/schemas/sitemap/0.9"> <url> <loc>https://srivenkateswaraborewells.github.io/WebSite/</loc> <lastmod>2025-08-24</lastmod> <changefreq>monthly</changefreq> <priority>1.0</priority> </url> </urlset>
4

Commit and push to GitHub

After creating or updating these files, commit and push them to your GitHub repository.

# Add files git add sitemap.xml robots.txt # Commit changes git commit -m "Add sitemap and robots.txt" # Push to GitHub git push origin main

Note: Replace "main" with your branch name if different

5

Wait for GitHub Pages to build

GitHub Pages typically takes a few minutes to build your site after pushing changes.

You can check the build status in your repository under the "Actions" tab.

6

Resubmit to Google Search Console

After verifying your files are accessible, go back to Google Search Console and resubmit your sitemap.

Enter the URL: https://srivenkateswaraborewells.github.io/WebSite/sitemap.xml
